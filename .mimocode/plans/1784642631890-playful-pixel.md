# Fix: Financial Distribution Data Not Saving on Edit

## Root Cause

**Race condition** between debounced save callbacks and Svelte reactive blocks in `FinanceTracker.svelte`.

The `updateMainCategory`, `updateGoal`, and `updateSavingsCategory` functions mutate `monthData` locally and schedule a debounced save (150ms delay). The debounced callback captures `monthData` **by reference** — not by value. If a Svelte reactive block overwrites `monthData` with stale store data before the callback fires, the stale data is saved.

### How the race happens:

1. User types → `updateMainCategory()` mutates `monthData.mainAccountCategories` locally, schedules debounced save (150ms)
2. `$tasks` or `$financialAnalyticsData` changes (e.g., timer tick)
3. Second reactive block (line 83-97) fires → calls `updateMonthData()` → changes `$financeData`
4. First reactive block (line 73-81) fires → `monthData = getMonthData(monthKey)` → reads **stale** store data (debounced save hasn't fired yet)
5. Debounced callback fires → reads `monthData.mainAccountCategories` (now stale) → saves stale data to store

### Affected functions in `FinanceTracker.svelte`:

| Function | Line | Field overwritten |
|---|---|---|
| `updateMainCategory` | 205-215 | `mainAccountCategories` |
| `updateGoal` | 239-249 | `monthGoals` |
| `updateSavingsCategory` | 275-285 | `savingsCategories` |

## Fix

**Capture values at scheduling time, not execution time.** In each debounced callback, snapshot the array to save into a local `const` before passing it to the closure.

### File: `src/finance/FinanceTracker.svelte`

#### 1. `updateMainCategory` (line 205-215)

```ts
function updateMainCategory(id: string, changes: Partial<FinanceCategory>) {
    monthData.mainAccountCategories = monthData.mainAccountCategories.map(c =>
      c.id === id ? { ...c, ...changes } : c
    );
    const categoriesToSave = monthData.mainAccountCategories; // snapshot
    debouncedUpdate(`main-${id}`, () => {
      updateMonthData(monthKey, {
        mainAccountCategories: categoriesToSave, // use snapshot
      });
      monthData = getMonthData(monthKey);
    });
  }
```

#### 2. `updateGoal` (line 239-249)

```ts
function updateGoal(id: string, changes: Partial<MonthGoal>) {
    monthData.monthGoals = (monthData.monthGoals || []).map(g =>
      g.id === id ? { ...g, ...changes } : g
    );
    const goalsToSave = monthData.monthGoals; // snapshot
    debouncedUpdate(`goal-${id}`, () => {
      updateMonthData(monthKey, {
        monthGoals: goalsToSave, // use snapshot
      });
      monthData = getMonthData(monthKey);
    });
  }
```

#### 3. `updateSavingsCategory` (line 275-285)

```ts
function updateSavingsCategory(id: string, changes: Partial<SavingsCategory>) {
    monthData.savingsCategories = monthData.savingsCategories.map(c =>
      c.id === id ? { ...c, ...changes } : c
    );
    const savingsToSave = monthData.savingsCategories; // snapshot
    debouncedUpdate(`savings-${id}`, () => {
      updateMonthData(monthKey, {
        savingsCategories: savingsToSave, // use snapshot

```
