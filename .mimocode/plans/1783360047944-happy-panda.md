# Мобильная адаптация — План

## Проблемы, которые нужно решить

1. **Touch-цели < 44px** — цветовые кружки (36px), кнопки дней недели (34px), кнопка привычки (28px), быстрые кнопки действий (26px), статус-кнопка (22px)
2. **Скрытые hover-элементы** — `.task-quick-actions` (opacity: 0) и кнопки edit/delete привычек невидимы на тач-устройствах без наведения
3. **Перемещение задач** — HTML5 drag-and-drop полностью неработоспособен на мобильных
4. **Контекстное меню календаря** — нет long-press альтернативы для правого клика
5. **Тултип при наведении на день** — `pointerover` ненадёжен на тач-устройствах
6. **Описание задачи** — скрыто на ≤480px полностью, хотя на больших телефонах (414px+) было бы полезно
7. **Модалка логов времени** — нет адаптации под клавиатуру
8. **Высота панелей vh** — при открытой клавиатуре 55vh/42vh может быть слишком много

### 4. Long-press для контекстного меню календаря
**Файл:** `src/ui/Calendar.svelte`

Добавить long-press обработчик через `touchstart`/`touchend`:

```svelte
<script>
  let longPressTimer = null;
  let longPressFired = false;
  const LONG_PRESS_MS = 500;
  const MOVE_THRESHOLD = 10;

  function handleTouchStart(date, event) {
    longPressFired = false;
    const touch = event.touches[0];
    const startX = touch.clientX;
    const startY = touch.clientY;

    longPressTimer = setTimeout(() => {
      longPressFired = true;
      onContextMenuDay(date, {
        pageX: startX,
        pageY: startY,
        preventDefault: () => {},
        stopPropagation: () => {},
      });
    }, LONG_PRESS_MS);
  }

  function handleTouchMove(event) {
    if (!longPressTimer) return;
    const touch = event.touches[0];
    // Отмена long-press при движении пальца (это скролл)
    clearTimeout(longPressTimer);
    longPressTimer = null;
  }

  function handleTouchEnd() {
    if (longPressTimer) {
      clearTimeout(longPressTimer);
      longPressTimer = null;
    }
  }
</script>
```

Проблема: `Day.svelte` из `obsidian-calendar-ui` привязывает свои обработчики напрямую, и мы не можем добавить touch-события через props.

**Решение:** Использовать делегирование событий на `#calendar-container` — добавить обработчики на контейнере после монтирования CalendarBase через `onMount`:

```svelte
import { onMount, afterUpdate } from "svelte";

let containerEl;

onMount(() => {
  setupTouchHandlers();
});

function setupTouchHandlers() {
  if (!containerEl) return;
  // Делегирование touch-событий на ячейках дней
  containerEl.addEventListener("touchstart", onDayTouchStart, { passive: false });
  containerEl.addEventListener("touchmove", onDayTouchMove, { passive: true });
  containerEl.addEventListener("touchend", onDayTouchEnd, { passive: true });
}

function findDayCell(target) {
  return target.closest(".day");
}
```

Но есть сложность: нам нужен доступ к `Moment` дате ячейки. `Day.svelte` хранит её в замыкании — она не доступна через DOM.

**Альтернативное решение (проще и надёжнее):**

Добавить `data-date` атрибут на ячейки дней через `obsidian-calendar-ui`. В `Day.svelte` (node_modules) добавить:
```svelte
<div class="day" ... data-date="{date.format('YYYY-MM-DD')}">
```

Затем в `src/ui/Calendar.svelte` — делегировать long-press через `data-date`:

```svelte
function onDayTouchStart(e) {
  const dayCell = e.target.closest('.day');
  if (!dayCell) return;
  const dateStr = dayCell.getAttribute('data-date');
  if (!dateStr) return;
  
  const touch = e.touches[0];
  let fired = false;
  
  longPressTimer = setTimeout(() => {
    fired = true;
    const moment = window.moment(dateStr, 'YYYY-MM-DD');
    onContextMenuDay(moment, {
      pageX: touch.clientX,
      pageY: touch.clientY,
      preventDefault: () => {},
      stopPropagation: () => {},
    });
  }, 500);
}
```

### 6. Календарный тултип на тач-устройствах
**Файл:** `src/view.ts`

На тач-устройствах тултип не появляется (pointerover ненадёжен). Вместо этого:
- При тапе на день — кратковременно показать тултип (2 сек), затем скрыть
- Или: показать количество задач в другом элементе (например, в шапке панели задач)

**Простейшее решение:** В `onHoverDay` — добавить проверку `app.isMobile`. Если mobile — не показывать тултип (информация о задачах уже видна в панели задач ниже календаря). Это уже фактически работает — тултип не появляется на мобилке.

Ничего не менять — текущее поведение приемлемо.
