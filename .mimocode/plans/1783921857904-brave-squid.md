# UI/UX Overhaul: Mobile + Desktop + Timer Persistence

## Summary
Comprehensive visual and interaction overhaul of the Obsidian Calendar Remastered plugin to improve mobile experience, reduce clutter on both platforms, and verify timer persistence on pause/resume.

---

## 1. Calendar — Visual Touch Feedback (Mobile)

**Problem**: No visual feedback when tapping a day on mobile. Desktop has hover effects (`transform: scale(1.1)`), but touch devices have hover disabled.

**File**: `styles.css` (lines 3099-3131, `@media (hover: none) and (pointer: coarse)`)

**Changes**:
- Add `:active` state for `.day` cells on touch devices: subtle scale-down + background color shift
- Add CSS `transition: transform 0.1s ease` for snappy response (shorter than the default 0.3s transition)
- Add `touch-action: manipulation` to prevent 300ms tap delay

```css
@media (hover: none) and (pointer: coarse) {
  #calendar-container .day {
    -webkit-tap-highlight-color: rgba(95, 153, 225, 0.3);
    transition: transform 0.1s ease, background 0.1s ease;
  }
  #calendar-container .day:active {
    transform: scale(0.92);
    background: var(--mcp-accent-dim);
  }
}
```

## 6. Timer Persistence — Verification (No Bug Found)

**Analysis of pause/resume flow**:

1. **Pause** (`pauseTaskTimer` in `stores.ts:198-243`):
   - Calls `stopTimer(id)` which returns a TimeLog with duration
   - Saves `totalWorkTime += log.duration`
   - Saves `pausedWorkTime = totalWorkTime` (snapshot at pause)
   - Saves `pausedAt = Date.now()`
   - Clears `timerStartedAt`
   - Sets `status = "paused"`
   - Calls `debouncedSave()` → persists to `calendar-data.json`

2. **Resume** (`startTaskTimer` in `stores.ts:130-153`):
   - Detects `task.status === "paused"` && `task.pausedAt` && `task.pausedWorkTime`
   - Starts a new timer session
   - Clears `pausedAt` and `pausedWorkTime`
   - Sets `timerStartedAt = Date.now()`

3. **On restart**: Timer is NOT in `activeTimers` (in-memory only), so no timer ticks. Task shows as "paused" with `totalWorkTime` preserved. User can resume manually.

**Conclusion**: Timer persistence works correctly. `totalWorkTime` is accumulated across pause/resume cycles and saved. On restart, the task starts paused with correct accumulated time. **No bug found.**
