# Mobile UI Overhaul — Полная переработка мобильного интерфейса

## Проблемы текущего мобильного UI

1. **Нет воздуха** — padding/margin слишком мелкие, элементы слипаются
2. **Шрифты слишком мелкие** — 7-8px (th), 9-10px (badges) — плохо читаются
3. **Touch targets нестабильные** — habit-check 26px, filter-btn 34px вместо 44px по HIG
4. **Панели сжимают экран** — task 40vh + habit 30vh = 70vh на маленьком экране
5. **Нет визуального разделения** — нет gap между элементами, нет визуальных группировок
6. **Календарь слишком плотный** — border-spacing 2px, ячейки 36px
7. **Нет scroll-snap на фильтрах** — горизонтальный scroll без snap
8. **Schedule button мелкий** — не привлекает внимание

## Стратегия изменений

**Файл**: `styles.css` (единственный файл с глобальными стилями)
**Breakpoints**: Оставляем существующие 3 tier (≤480px phone, ≤360px small phone, touch devices)
**Подход**: Увеличиваем padding, gap, font-size; добавляем whitespace; сохраняем glassmorphism
