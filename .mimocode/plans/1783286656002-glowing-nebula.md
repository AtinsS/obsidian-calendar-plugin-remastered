# Task Tracker Pro - Implementation Plan

## Phase 0: Critical Bug Fix — Badge Not Showing

**Root cause**: `taskDotSource.ts` returns `dataAttributes: { "task-count": N }` which becomes HTML attribute `task-count="N"`. But CSS selector `[data-task-count]` expects `data-task-count="N"`. The `data-` prefix is missing.

### Files to modify

1. **`src/task-tracker/taskDotSource.ts`** — Change key from `"task-count"` to `"data-task-count"`. Add all-completed and overdue states:
   - If all tasks completed → set class `all-completed`, attribute value = `"✓"`
   - If has overdue tasks → set class `has-overdue`
   - Count display: 0=no badge, 1-9=number, 10+="9+"

2. **`styles.css`** — Fix CSS specificity and add badge states:
   - `#calendar-container .calendar` instead of `.calendar` (beats library's `.svelte-pcimu8` scoped selector)
   - `.day.all-completed[data-task-count]::after` — green badge with checkmark
   - `.day.has-overdue[data-task-count]::after` — red pulsing badge
   - Add tooltip CSS for `[data-task-tooltip]`
   - Fix `::after` to not use `display: flex` (invalid on pseudo-elements)

## Phase 2: Habit Tracker (NEW module)

### 2.1 New files

```
src/habit-tracker/
├── types.ts           — IHabit, IHabitLog interfaces
├── stores.ts          — Svelte stores + CRUD
├── HabitPanel.svelte  — Main habit list UI
├── HabitItem.svelte   — Individual habit row with streak
├── HabitModal.ts      — Create/edit habit modal (Obsidian Modal)
├── Heatmap.svelte     — GitHub-style 365-day heatmap
└── habitSource.ts     — ICalendarSource for habit badge data
```

### 2.2 Types (`src/habit-tracker/types.ts`)

```typescript
interface IHabit {
  id: string;
  title: string;
  icon: string;           // emoji
  color: string;          // hex color
  frequency: 'daily' | 'weekly' | 'custom';
  customDays?: number[];  // for custom frequency
  projectId?: string;
  targetCount?: number;   // default 1
  archived: boolean;
  createdAt: string;
}

interface IHabitLog {
  id: string;
  habitId: string;
  date: string;           // YYYY-MM-DD
  completed: boolean;
  count?: number;
  completedAt?: string;
}

interface IHabitTrackerData {
  habits: IHabit[];
  habitLogs: IHabitLog[];
  version: number;
}
```

### 2.3 Stores (`src/habit-tracker/stores.ts`)

- `habits`, `habitLogs` writable stores
- `initHabitStores(plugin)` — loads from data.json key `"habitTracker"`
- CRUD: `addHabit`, `updateHabit`, `removeHabit`
- `logHabitCompletion(habitId, date, count?)` — toggle or increment
- `getHabitLogsForHabit(habitId)` — all logs for a habit
- `getHabitLogsForDate(date)` — all completions on a date
- `calculateStreak(habitId)` — current streak count
- `calculateLongestStreak(habitId)` — longest ever streak
- `getCompletionRate(habitId, days)` — % over last N days
- `debouncedSave()` — 300ms debounce

### 2.4 HabitPanel.svelte

- Header with "Привычки" title + overall stats
- List of active habits grouped or flat
- Each habit: icon + title + streak badge + completion toggle button + heatmap mini
- "Добавить" button opens HabitModal
- Archived habits hidden by default, toggle to show

### 2.5 HabitItem.svelte

- Icon, title, streak fire emoji + count
- Toggle button (checkmark when completed today)
- Mini 7-day bar (last 7 days, filled/empty)
- Click to open details / edit
- Drag support for reordering

### 2.6 HabitModal.ts

- Obsidian Modal with fields: title, icon (emoji picker or text), color (same grid as projects), frequency, custom days (if custom), target count, project link

### 2.7 Heatmap.svelte

- 365-day grid (52 weeks x 7 days), GitHub-style
- Color intensity based on completion count (0-4+ levels)
- Tooltip on hover: date + completions count
- Summary stats below: total days, current streak, longest streak

### 2.8 habitSource.ts (ICalendarSource)

- Returns `data-habit-count` attribute for days with habit logs
- Adds `has-habit-logs` CSS class
- Badge shows total habit completions for the day

### 2.9 Calendar integration (`src/view.ts`)

- Add `habitSource` to sources array
- Mount HabitPanel below TaskPanel

### 2.10 Settings (`src/settings.ts`)

- Add `showHabitTracker: boolean` setting

## Phase 3: Pomodoro Timer (NEW module)

### 3.1 New files

```
src/pomodoro/
├── types.ts              — IPomodoroSession, IPomodoroSettings
├── stores.ts             — Timer state + session log stores
├── PomodoroWidget.svelte — Full timer UI
├── CircularProgress.svelte — SVG circular progress ring
├── PomodoroSettingsModal.ts — Timer configuration modal
└── sound.ts              — Web Audio API notification sounds
```

### 3.2 Types

```typescript
type PomodoroMode = 'work' | 'short-break' | 'long-break';

interface IPomodoroSession {
  id: string;
  taskId?: string;
  habitId?: string;
  type: PomodoroMode;
  duration: number;          // in minutes
  startedAt: string;
  completedAt?: string;
  interrupted: boolean;
}

interface IPomodoroSettings {
  workDuration: number;      // default 25
  shortBreakDuration: number; // default 5
  longBreakDuration: number;  // default 15
  longBreakInterval: number;  // default 4
  autoStartBreaks: boolean;
  autoStartPomodoros: boolean;
  soundEnabled: boolean;
  soundVolume: number;        // 0-1
}
```

### 3.3 Stores

- `pomodoroSettings` writable store
- `activePomodoro` writable store: `{ mode, timeLeft, totalSeconds, sessionsCompleted, isRunning, taskId? }`
- `pomodoroSessions` writable store — all completed sessions
- `startPomodoro(taskId?)` — starts timer interval
- `pausePomodoro()` / `resumePomodoro()` / `resetPomodoro()`
- `completeSession()` — save session, switch mode, play sound, show notice

### 3.4 PomodoroWidget.svelte

- Mode indicator (Работа / Короткий перерыв / Длинный перерыв)
- CircularProgress SVG (stroke-dashoffset animation)
- Time display (MM:SS)
- Start / Pause / Reset buttons
- Session dots: N filled + M empty (sessions completed / long break interval)
- Task binding dropdown (select from active tasks)
- Expand/collapse

### 3.5 CircularProgress.svelte

- SVG circle with stroke-dasharray/dashoffset
- Color changes by mode: work=accent, break=green
- Smooth CSS transition on progress

### 3.6 sound.ts

- Web Audio API: generate simple tones (no external files needed)
- Work complete: ascending chime (C5-E5-G5)
- Break complete: descending tone
- Volume control from settings

### 3.7 Mount in view.ts

- Add PomodoroWidget above TaskPanel (below calendar)
- Setting: `showPomodoro: boolean`

## Phase 4: Notification System (NEW module)

### 4.1 New files

```
src/notifications/
├── types.ts             — IReminder interface
├── NotificationService.ts — Scheduler + Obsidian Notice integration
└── ReminderModal.ts     — Set reminder on task/habit
```

### 4.2 Types

```typescript
interface IReminder {
  id: string;
  type: 'task' | 'habit' | 'pomodoro';
  entityId: string;
  title: string;
  message: string;
  scheduledFor: string;    // ISO datetime
  repeat: 'none' | 'daily' | 'weekly';
  triggered: boolean;
  createdAt: string;
}
```

### 4.3 NotificationService.ts

- `init(plugin)` — loads reminders, starts 30s check interval
- `addReminder(reminder)` / `removeReminder(id)`
- `checkReminders()` — called every 30s, fires `new Notice()` for due reminders
- `schedulePomodoroNotification(seconds)` — one-shot timer for pomodoro completion
- Cleanup on plugin unload

### 4.4 ReminderModal.ts

- Add to TaskModal: "Напоминание" dropdown (нет / за 15 мин / за 30 мин / за 1 час / в заданное время)
- Creates IReminder linked to taskId

### 4.5 Settings

- `notificationsEnabled: boolean`
- `quietHoursStart: string` (HH:MM)
- `quietHoursEnd: string` (HH:MM)
